## Knowledge Base (ICKB) Usage [STATIC]

The ICKB at `http://192.168.51.223:8001/kb` has FTS5 search and topic
indexing. Use it for:

- **In-flight troubleshooting findings** → post as relay messages
  (`POST /write`) so other Claudes see them in real time
- **Resolved issues** → post to ICKB (`POST /kb/post`) with
  category=`troubleshooting`. Include symptom, root cause, fix,
  verification
- **Architecture and conventions** → ICKB with category=`patterns` or
  `conventions`
- **Secrets** → store API keys, tokens, and credentials in ICKB. **The
  ICKB is currently unencrypted and LAN-readable** — an encryption layer
  is a stated priority for the relay maintainers. Until encryption is
  added, prefer storing references (host + table + row, host + env var
  name, host + file path) over secret values when possible. When you do
  store a value, tag the entry `category=secrets` so it can be migrated
  during the encryption rollout

Search before working:
```bash
curl -sS 'http://192.168.51.223:8001/kb/search?q=<terms>'
```
