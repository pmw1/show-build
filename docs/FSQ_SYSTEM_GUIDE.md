## Overview

The FSQ system creates full-screen quote graphics for video production. It provides intelligent quote processing, automatic attribution extraction, LLM-powered split recommendations, and real-time preview with accurate scaling to match final 1920x1080 output.

### Key Features
- **Automatic Attribution Detection** - Extracts attribution from pasted quotes
- **Intelligent Quote Splitting** - LLM analyzes quotes and recommends splits for readability
- **Dual Preview Mode** - Compare LLM split vs original side-by-side
- **Manual Split Controls** - Click to mark custom split points
- **Realistic Previews** - 16:9 aspect ratio with scaled text matching final output
- **Quote Normalization** - Automatic fixing of nested/smart quotes
- **Database-Backed Settings** - All preferences persist across sessions

## Basic Workflow

### 1. Open FSQ Modal
- Click "FSQ" button in script editor cue toolbar
- Modal opens with empty quote field

### 2. Enter/Paste Quote
```
"The only thing we have to fear is fear itself"
- Franklin D. Roosevelt
```

**Automatic Processing:**
- Attribution detected and extracted → moved to "Source/Attribution" field
- Exterior quotes stripped (if enabled in settings)
- Slug auto-generated from first 3 words: `the-only-thing`
- LLM analysis starts after 2-second typing pause

### 3. Review Preview
- 16:9 video preview shows quote as it will render
- Adjust alignment, font, and size with live preview
- Text scales to match 1920x1080 final output

### 4. Handle LLM Recommendations
If quote is too long:
- **LLM Split Section** (purple border) shows multi-part preview
- **Original Section** (black border) shows unsplit version
- **Manual Split Section** (blue border) allows custom splits

### 5. Insert
- Click "ACCEPT LLM SPLIT" to insert multi-part quotes
- Click "REJECT LLM / INSERT AS-IS" to insert original
- Each part gets unique AssetID and slug

## Automatic Features

### Auto-Attribution Extraction

**Triggers:** On paste or text input
**Runs:** Before exterior quote stripping

**Supported Patterns:**

**Pattern 1: Newline + Dash (Most Common)**
```
"Quote text here"
- Person Name
```

**Pattern 2: Newline + Em-Dash (No Space)**
```
"Quote text here"
—Person Name
```

**Pattern 3: Same Line**
```
"Quote text here" - Person Name
```

**Pattern 4: Newline Only (No Dash)**
```
"Quote text here"
Person Name
```

**Validation Rules:**
- Attribution: 2-100 characters
- Quote: >10 characters, longer than attribution
- Attribution must start with capital letter or dash
- Only extracts if attribution field is empty

**Result:**
- Quote cleaned and placed in quote field
- Attribution placed in source field
- "Include Attribution" enabled automatically
- Console logs extraction details

### Auto-Slug Generation

**Triggers:** On quote input (if slug not manually edited)

**Process:**
1. Takes first 3 words of quote
2. Converts to lowercase
3. Removes punctuation
4. Replaces spaces with hyphens
5. Removes duplicate/trailing hyphens

**Example:**
```
Quote: "I think, therefore I am"
Slug:  i-think-therefore
```

**Stop Auto-Generation:**
- Manually edit slug field
- Flag `slugAutoGenerated` set to `false`

### Auto-Source Population

**Triggers:** On modal open

**Priority:**
1. Last submitted source in current session
2. Most recent source from episode rundown
3. Empty (user must enter)

**Autocomplete:**
- Dropdown shows all sources from current episode
- Type to filter
- Select from history

## LLM-Powered Features

### Intelligent Quote Analysis

**Model:** Qwen2.5-Coder:7b (via Ollama)
**Trigger:** 2 seconds after typing stops
**Minimum Length:** 20 characters

**Analysis Steps:**

1. **Nested Quote Normalization**
   - Detects nested/smart quotes
   - Converts to American English style (outer: double, inner: single)
   - Auto-applies if setting enabled

2. **Split Recommendation**
   - Analyzes quote length vs screen capacity
   - Considers sentence boundaries
   - Suggests optimal split points
   - Creates multi-part segments

**Settings (Database-Backed):**
```javascript
{
  fsq_max_lines: 5,              // Max lines per screen
  fsq_chars_per_line: 50,        // Characters per line
  fsq_min_second_screen: 80,     // Min chars for 2nd screen
  fsq_split_strategy: 'smart',   // 'smart' or 'balanced'
  fsq_balance_threshold_percent: 30,  // Max imbalance %
  fsq_prefer_sentence_boundaries: true,
  fsq_allow_mid_sentence_split: false,
  fsq_overflow_handling: 'multi_segment'
}
```

### Visual Feedback

**Field Border Colors:**
- **Purple Glow + Pulsing:** Analyzing
- **Green Glow:** Approved (no split needed)
- **Red Glow:** Needs review (split recommended)

**AI State Indicator:**
- **🧠 Purple Spinning:** Analyzing
- **✓ Green:** Approved
- **⚠ Red:** Needs review
- **🤖 Blue:** Auto-processed

## Preview Modes

### Single Preview Mode
**When:** No split recommendations

**Display:**
- One 16:9 preview container
- Live video background
- Text overlay with gradient
- Controls: Alignment, Font, Size
- Render mode selection

### Dual Preview Mode
**When:** LLM recommends split OR manual split applied

**Three Sections:**

#### 1. LLM Recommended Split (Purple Dotted Border)
- Title: "LLM RECOMMENDED SPLIT"
- Multiple 16:9 previews stacked vertically
- Each labeled "PART 1/3", "PART 2/3", etc.
- Shared controls affect ALL parts equally
- Purple "ACCEPT LLM SPLIT" button

#### 2. Original Quote (Black Dotted Border)
- Title: "ORIGINAL (UNSPLIT)"
- Single 16:9 preview
- Independent controls (different from split)
- Grey "REJECT LLM / INSERT AS-IS" button

#### 3. Manual Split (Blue Dotted Border)
- Title: "MANUAL SPLIT"
- Clickable quote text display
- Mark split points by clicking
- Chip badges show each marker
- Blue "APPLY MANUAL SPLIT" button

## Quote Formatting Settings

**Location:** Settings → Generation → Quote Formatting Rules

### 1. Strip Exterior Quotes During Processing
**Default:** Yes
**Effect:** Removes surrounding `"` or `'` from input text

**Example:**
```
Input:  "Quote text here"
Output: Quote text here
```

### 2. Regenerate Exterior Quotes at Render
**Default:** No (NEW - quotes no longer added back)
**Effect:** Adds `"` around quote in final render

**Use Case:** Enable if you want quotes to appear in graphic

### 3. Normalize Interior Quotes to Single Quotes
**Default:** Yes
**Effect:** Converts nested quotes to American English style

**Example:**
```
Input:  He said "I think it's "great""
Output: He said "I think it's 'great'"
```

### 4. Attribution Dash Style
**Default:** Regular Dash
**Options:**
- **Regular Dash:** `— ` (space after)
- **Em Dash:** `—` (no space)
- **None:** `` (no dash, just space)

**Rendered As:**
```
Regular: — Franklin D. Roosevelt
Em Dash: —Franklin D. Roosevelt
None:    Franklin D. Roosevelt
```

## Attribution Handling

### Include/Omit Toggle
- **Include:** Attribution appears below quote
- **Omit:** No attribution rendered (source still saved to cue)

### Attribution Field
- Text input with autocomplete
- Combobox allows selection from episode history
- Required when "Include" is selected
- Optional when "Omit" is selected

### Auto-Population
1. **On Paste/Input:** Extracted from quote text
2. **On Modal Open:** Last used source in session
3. **From History:** Most recent in episode rundown

### Styling
- Font: Matches quote font family
- Size: Smaller than quote text
- Position: Below quote, centered/aligned
- Shadow: Text shadow for readability

## Split Quote Management

### LLM Split Workflow

1. **LLM Analyzes Quote**
   - Determines optimal split points
   - Creates array of segments
   - Stores in `splitRecommendations`

2. **Dual Preview Appears**
   - Purple section shows all split parts
   - Black section shows original
   - Blue section allows manual override

3. **User Chooses**
   - **Accept LLM Split:** Uses `splitQuoteStyle`, `splitFontFamily`, `splitFontSize`
   - **Reject LLM:** Uses `quoteStyle`, `fontFamily`, `fontSize`
   - **Manual Split:** Apply custom markers, use split controls

### Multi-Part Insertion

**Accept LLM Split:**
```javascript
// Each segment gets:
{
  type: 'FSQ',
  assetId: 'FSQ-0244-001',      // Unique per part
  slug: 'quote-title-1x3',       // Part indicator
  quote: 'First segment text',
  style: splitQuoteStyle,        // ALL parts same style
  fontFamily: splitFontFamily,
  fontSize: splitFontSize,
  duration: '00:00:05',
  part: '1x3'                    // Part 1 of 3
}
```

**Inserted Sequentially:**
- Part 1 → Part 2 → Part 3
- Each gets consecutive AssetID
- Each gets `<p class="josh"></p>` after for narration

## Manual Splitting

### How to Use

1. **Quote Text Entered**
   - LLM may or may not recommend split
   - Manual Split section always visible in dual mode

2. **Click to Mark Split Points**
   ```
   Click here ↓
   "First part of quote. Second part of quote."
                         ^
   ```

3. **Split Markers Appear**
   - Blue chips show: `Split 1 @ 25`
   - Click X to remove marker
   - Markers sorted by position

4. **Apply Manual Split**
   - Click "APPLY MANUAL SPLIT"
   - Creates segments from markers
   - Treated as LLM recommendation
   - Dual preview appears with manual segments

### Technical Details

**Click Detection:**
```javascript
markManualSplit() {
  const selection = window.getSelection()
  const offset = selection.getRangeAt(0).startOffset
  manualSplitPoints.push(offset)
}
```

**Segment Creation:**
```javascript
segments = []
lastIndex = 0
for (splitPoint in manualSplitPoints) {
  segment = quote.substring(lastIndex, splitPoint)
  segments.push(segment)
  lastIndex = splitPoint
}
// Add final segment
segments.push(quote.substring(lastIndex))
```

**AI State:**
- Marked as `aiState: 'auto'` (user-generated)
- `aiGenerationInfo.model: 'Manual Split'`

## Technical Details

### Preview Scaling

**Problem:** Preview container is responsive, but text must match 1920px output

**Solution:**
```javascript
// Calculate font size as percentage of container width
const fontSizePercent = (fontSize / 1920) * 100

quoteTextStyle() {
  return {
    fontSize: `${fontSizePercent}%`  // Scales with container
  }
}
```

**CSS:**
```css
.quote-preview-container {
  aspect-ratio: 16 / 9;  /* Lock to 16:9 */
  width: 100%;
}

.quote-preview {
  font-size: 16px;  /* Base for percentage calculation */
}

.quote-text {
  /* fontSize from computed property = (25/1920)*100 = 1.302% */
  /* At 700px container: 700 * 1.302% = 9.1px */
  /* At 1920px container: 1920 * 1.302% = 25px ✓ */
}
```

  fsq_default_font_size: 25,
  fsq_include_attribution_by_default: true,

### AssetID Generation

**Format:** `FSQ-{episode}-{sequence}`
**Example:** `FSQ-0244-001`, `FSQ-0244-002`, `FSQ-0244-003`

**Sequential Parts:**
```javascript
for (let i = 0; i < segments.length; i++) {
  const assetId = generateAssetId()  // FSQ-0244-001, FSQ-0244-002, etc.
  const slug = `${baseSlug}-${i+1}x${segments.length}`
}
```

## Troubleshooting

### Quote Not Auto-Analyzing
- **Check:** Minimum 20 characters
- **Check:** 2-second pause after typing
- **Check:** Ollama service running at `192.168.51.197:11434`

### Attribution Not Auto-Extracting
- **Check:** Attribution on new line or after dash
- **Check:** Source field is empty (won't override)
- **Check:** Attribution starts with capital letter
- **Check:** Quote >10 characters

### Preview Text Size Wrong
- **Check:** fontSize setting (in pixels)
- **Check:** Preview container has 16:9 aspect ratio
- **Check:** Browser supports `aspect-ratio` CSS property
- **Fallback:** Modern browsers only (Chrome 88+, Firefox 89+)

### Split Preview Not Showing
- **Check:** `splitRecommendations` array has >1 items
- **Check:** LLM analysis completed
- **Check:** Modal in dual preview mode

### Video Background Not Playing
- **Check:** `fsq_background_video` path in settings
- **Check:** Video file exists at path
- **Check:** Browser autoplay policy (muted videos allowed)

## Best Practices

### Quote Entry
1. Paste with attribution on new line
2. Let auto-extraction work
3. Wait 2 seconds for LLM analysis
4. Review split recommendations

### Attribution
- Use consistent formatting: "First Last, Title"
- Use autocomplete to maintain consistency
- Include titles when relevant: "John Doe, CEO"

### Splitting
- Trust LLM recommendations for readability
- Use manual split for creative control
- Preview all parts before accepting
- Ensure each part makes sense independently

### Styling
- Use consistent font across episode
- Adjust size based on quote length
- Preview at different screen sizes
- Test final output at 1920x1080
